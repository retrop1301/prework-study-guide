HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML attributes are always defined in the beginning element, never the closing element
  • The < link > element connects an external resource to the HTML document
  • The < script > element embeds client-side scripts in the HTML document
  • the < meta > element defines the charset, or character set

CSS

  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • the most common way to style a webpage is by using an external CSS style sheet
  • Declarations contain two important components: the CSS property we want to apply and the value of the property
  • The < img > selector assigns property values to any image element in the markup
  • The display property is assigned with the block value, which assigns each image to start on a new line

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: add command adds modifications in the current working branch to the staging area. The -A flag indicates that we want to add all changes
  • pull command to receive a branch's modifications into the local environment
  • push to push the changes we have locally to our remote GitHub branch

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed
  • A for loop contains: the starting point for our loop, conditions, and iterations of predictable patterns
